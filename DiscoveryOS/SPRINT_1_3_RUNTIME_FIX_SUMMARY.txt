WHAT WAS BROKEN

1. npm run start:dev crash with:
   "TypeError: Cannot read properties of undefined (reading 'fileExists')"

2. npm run dev script did not exist

3. @nestjs/config peer dependency conflict with NestJS 11

================================================================================
ROOT CAUSES & FIXES
================================================================================

ROOT CAUSE #1: TypeScript 7.0.2 + ts-node Incompatibility
  Problem: TypeScript 7 breaks ts-node with moduleResolution changes
  Error: ts-node can't initialize TypeScript system
  Fix: Downgrade typescript to 5.9.3

ROOT CAUSE #2: Wrong moduleResolution Setting
  Problem: "bundler" not supported by ts-node 10.x
  Error: Configuration not read properly
  Fix: Change to "moduleResolution": "node"

ROOT CAUSE #3: @nestjs/config 3.3.0 Peer Dependency
  Problem: Requires NestJS 8-10, we have 11.1.28
  Error: npm install fails without --legacy-peer-deps
  Fix: Downgrade @nestjs/config to 3.2.0

ROOT CAUSE #4: Incompatible @types/node 26.1.1
  Problem: Too new for TypeScript 5.9.3
  Error: Type resolution conflicts
  Fix: Downgrade to @types/node 20.19.43

ROOT CAUSE #5: Missing npm run dev Script
  Problem: Only start:dev existed, not dev
  Error: "npm: Unknown script 'dev'"
  Fix: Add "dev" script to package.json
